# FR-224 - BayesSeg Bayesian Passage Segmentation

## Overview
BayesSeg models a document as a sequence of segments, each generated by its own multinomial language model whose parameters are drawn from a Dirichlet prior. Maximum-a-posteriori segmentation under this generative model is found exactly by dynamic programming, and the resulting boundary log-likelihood gives a principled, calibratable measure of segment cohesion. Of the four segmentation algorithms (FR-221 TextTiling, FR-222 C99, FR-223 dotplotting, FR-224 BayesSeg), BayesSeg is the only fully-Bayesian one and is the most robust on short or noisy documents. Used as a small additive bonus on the candidate's passage-relevance term.

## Academic source
**Eisenstein, Jacob; Barzilay, Regina (2008).** "Bayesian Unsupervised Topic Segmentation." *Proceedings of the 2008 Conference on Empirical Methods in Natural Language Processing (EMNLP 2008)*, Honolulu, Hawaii, pp. 334-343. URL: `https://aclanthology.org/D08-1035/`. Defines the generative model (Dirichlet–multinomial per segment), the MAP-by-DP inference, and the segment-cohesion log-likelihood used in the formula below.

## Formula
From Eisenstein & Barzilay (2008), §2 (generative model) and §3 (MAP via DP):

```
Generative model. Document = sequence of n words W = w_1..w_n. Hidden segmentation
B = (b_0 = 0, b_1, b_2, ..., b_K = n) cuts W into K segments. Each segment k is
assigned a multinomial θ_k over the vocabulary V, with θ_k ~ Dirichlet(α).

  P(W | B, α) = Π_{k=1..K} ∫ P(words(segment k) | θ_k) · P(θ_k | α) dθ_k

For each segment, the marginal under Dirichlet–multinomial conjugacy (Polya
distribution) is:

  P(words_k | α) = Γ(Σ_v α_v) / Γ(N_k + Σ_v α_v) · Π_v Γ(c_{k,v} + α_v) / Γ(α_v)

where:
  c_{k,v} = count of word v in segment k
  N_k     = total tokens in segment k
  Γ       = gamma function

Penalty for over-segmentation:
  P(B | K, λ) ∝ exp(−λ · K)             (default λ = log(n) — minimum description length)

MAP segmentation by dynamic programming:
  V[0] = 0
  V[j] = max_{i < j}  ( V[i] + log P(w_{i+1..j} | α) − λ )       for j = 1..n

  B* = backtrack from V[n]                                       (best K and cuts)

Per-passage signal for an insertion point at token index t:
  seg* = MAP segment containing t
  cohesion(seg*) = ( log P(seg* | α) ) / N_{seg*}                (per-token log-likelihood)

  signal(t) = clamp( (cohesion(seg*) − μ_cohesion) / σ_cohesion · 0.25 + 0.5,  0,  1 )
                                                                  (z-score → [0,1])
```

Where:
- `α ∈ ℝ_+^|V|` is the Dirichlet prior (default symmetric `α_v = 0.5`)
- bigger `cohesion` = higher per-token log-likelihood = more vocabulary coherence
- `signal ∈ [0, 1]` — `0.5` is the document-mean cohesion, with shape parameter `0.25` keeping `±2σ` inside the unit interval

## C++ implementation
- File: `backend/extensions/bayesseg.cpp`
- Entry: `double bayesseg_cohesion(const uint32_t* token_ids, int n_tokens, double alpha, double lambda, int max_K, int insertion_token_idx);`
- Complexity: `O(n²)` for the DP table; `O(n · |V_seg|)` per segment for log-likelihood
- Thread-safety: pure function on token-id buffer; no shared state
- SIMD: `_mm256_*` for the per-segment vocabulary-count log-gamma reduction
- Uses `std::lgamma` for stable log Γ computation; precomputes `lgamma(α)` once
- Builds against pybind11 alongside passage-level extensions (FR-053 family)

## Python fallback
`backend/apps/pipeline/services/bayesseg.py::compute_bayesseg_signal(...)` — used when the C++ extension is unavailable; reuses tokenised document already built in FR-053. Uses `scipy.special.gammaln` for stable log Γ.

## Diagnostics

- MAP segment id and bounds (start..end token index)
- All MAP segment cohesion values (sparkline)
- DP-decided segment count `K*` and the MDL penalty `λ` used
- Document-level mean and stdev of cohesion (the z-score basis)

## Edge cases & neutral fallback
- Document below `min_tokens` → neutral `0.5`, flag `below_min_tokens`
- Single-segment MAP solution (whole doc one topic) → cohesion uses doc-mean baseline; signal `0.5`, flag `single_segment`
- All-identical-token document → cohesion well-defined but degenerate; signal `0.5`, flag `degenerate_vocab`
- Insertion token past `n_tokens` → segment = last segment, no flag
- NaN / Inf log Γ → cell replaced with neutral `0.5`, flag `nan_clamped`

## Minimum-data threshold
`≥ 200` tokens before the Dirichlet–multinomial likelihood is statistically meaningful; below this returns neutral `0.5` with flag `below_min_tokens`.

## Budget
Disk: <2 MB (precomputed `lgamma` table)  ·  RAM: <80 MB (`n²` DP table + per-segment count buffers, capped at `n = 20 000`)

## Scope boundary vs existing signals
FR-224 does NOT overlap with FR-221 (TextTiling), FR-222 (C99), or FR-223 (dotplotting) — all four are passage-segmentation methods with different mathematical bases. BayesSeg is uniquely Bayesian (proper generative model with calibrated likelihoods). Operators run all four in parallel and FR-018 auto-tuning blends or selects the best. It does not overlap with FR-053 (passage-level relevance) which measures destination match, not host-passage cohesion.
